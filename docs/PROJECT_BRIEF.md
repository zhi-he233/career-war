# 职业互怼 PROJECT_BRIEF

## 项目简介

这是一个 H5 联机回合制桌游，游戏名暂定为《职业互怼》。玩家通过手机浏览器进入房间，选择职业，轮流投骰，服务器根据职业技能自动结算伤害、治疗、护盾、死亡和胜利。

## 技术栈

* 前端：Vue 3 + TypeScript + Vite
* 后端：Node.js + Express + Socket.IO
* 规则共享：shared 目录
* 当前不使用数据库，房间和玩家状态暂存在服务器内存中

## 目录结构

* client/：前端页面、组件和样式
* server/：后端服务、Socket.IO 房间和事件
* shared/：前后端共用类型、职业数据和规则引擎
* docs/：项目说明、TODO、规则记录

## 核心原则

1. 服务器负责投骰和结算。
2. 前端只负责显示和发送玩家操作。
3. 前端不能自行决定伤害、治疗、死亡或胜负。
4. 游戏规则逻辑尽量放在 shared/engine.ts。
5. 职业数据尽量放在 shared/characters.ts。
6. Socket.IO 只负责通信，不要把大量规则写死在 socket 事件里。
7. 每次行动前应保存快照，方便以后实现“逆流”等撤销类职业。
8. 手机竖屏优先。
9. 当前阶段不做数据库、不做账号、不做小程序、不做排行榜、不做美术资源。

## 当前回合定义

1回合指：从该角色本次行动开始，到该角色下一次行动开始之前。

## 默认战斗规则

如果职业没有特殊说明，投到几点，就造成几点伤害。

角色血量小于等于 0 时死亡。
死亡角色跳过行动。
只剩一名存活玩家时，该玩家胜利。
护盾优先扣除，除非技能写明无视护盾。
