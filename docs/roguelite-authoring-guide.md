# 肉鸽内容填写说明

这几份文档是给策划内容用的。你只需要改 `docs/` 里的 Markdown 表格和说明，不需要写代码。

## 你可以写什么

* 问号事件剧情
* 事件选项
* 事件奖励
* 事件代价
* 商店商品
* 商店价格
* 休息点效果
* 精英 / Boss 主题想法
* 地图路线建议

## 你不要写什么

* TypeScript 代码
* Vue 代码
* 服务端代码
* engine 结算逻辑
* 复杂公式
* 随机算法

## 怎么填

1. 每一行写一个事件、商品、休息行为或路线建议。
2. `id` 用英文小写和下划线，例如 `cycle_bell`。
3. 文案可以写中文，越具体越好。
4. 效果和代价必须写具体数值，禁止使用模糊词。参照各文档头部的数值标尺表。
5. 不要删除表头。如果暂时没有内容，可以新增一行写"待定"。
6. 如果不确定具体数值，先标"候选"并在备注里写"待定数值"，不要写"少量/中量/大量"。

## 数值标尺速查

| 模糊词（禁止使用） | 应写为 |
| - | - |
| 少量生命损失 | 失去 3 生命 |
| 中量生命损失 | 失去 6 生命 |
| 大量生命损失 | 失去 10 生命 |
| 少量回复 | 回复 5 生命 |
| 中量回复 | 回复 10 生命 |
| 大量回复 | 回复 15 生命 |
| 少量金币 | 20 金币 |
| 中量金币 | 45 金币 |
| 大量金币 | 80 金币 |
| 少量护盾 | 4 护盾 |
| 中量护盾 | 8 护盾 |
| 大量护盾 | 12 护盾 |
| 敌人更强 | 下一场敌人生命 +6，伤害 +1 |
| 战斗更危险 | 后续 3 场敌人伤害 +1 |
| 高品质奖励 / 较好奖励 | 从稀有奖励池三选一 |
| 普通成长 | 从普通成长池三选一 |
| 角色技能奖励 | 从角色技能池三选一 |
| Boss 能力奖励 | 从 Boss 能力池三选一 |
| 第一次攻击伤害提高 | 下一场第一次攻击伤害 +4 |

## 推荐填写顺序

1. 先读 `docs/roguelite-room-types.md`，理解每种房间是什么。
2. 再填 `docs/roguelite-events.md` 的问号事件。
3. 然后填 `docs/roguelite-shop.md` 和 `docs/roguelite-rest-sites.md`。
4. 最后在 `docs/roguelite-map-rules.md` 里写路线节奏建议。
