# 肉鸽模式策划表

> 本文档给策划和非程序编辑阅读。游戏运行时不解析 Markdown。
> 修改本表后，需要由 AI 或程序员同步到 `shared/src/data/rogueliteBalance.ts`。

## 编辑原则

- 只影响 `gameMode === "pve_roguelite"`。
- 不修改普通职业数值，不修改召唤师技能数值。
- 不影响 classic、duo_2v2、pve_1v1。
- Boss 强度优先来自机制，不优先堆血。
- 数值改动需要同步 TS 数据源，不能只改本文档。

## 朋友编辑规则

- 可以改数值、名称、说明、权重、出现关卡和备注。
- 可以新增“候选”敌人、奖励、Boss、玩法想法，先写进候选区或在状态列标记为“候选”。
- “状态”只能填写：启用、候选、废弃。
- 不要随便改 `id` / `type`。它们是程序识别用的稳定字段，改名可能导致旧逻辑找不到配置。
- 不要删除表头。AI 同步 TS 数据时会按表头理解每一列含义。
- “启用”表示会同步进 `shared/src/data/rogueliteBalance.ts`，并可能进入游戏。
- “候选”只是想法，不会自动进游戏。
- “废弃”只保留历史记录，不会进游戏。
- Markdown 不会被游戏运行时直接读取。
- 改完 Markdown 后，需要让 AI 同步 `shared/src/data/rogueliteBalance.ts`。
- 同步 TS 数据后必须运行 `npm run build`。
- 如果不确定一个改动会不会影响战斗结算，先写成“候选”，不要直接标成“启用”。

## 关卡奖励节奏

| 状态 | 触发关卡 | 奖励类型 | 目标 |
| --- | --- | --- | --- |
| 启用 | 第 1 关胜利 | 基础奖励三选一 | 给生命、护盾、伤害、护甲、治疗等基础成长 |
| 启用 | 第 2 关胜利 | 流派启动奖励三选一 | 尽量来自不同 tag，让玩家开始定方向 |
| 启用 | 第 3 关 Boss 胜利 | 核心技能奖励三选一 | 至少出现一个关键技能 |
| 启用 | 第 4-6 关胜利 | 倾向已有 tag | 略微提高同流派奖励出现概率 |
| 启用 | 第 15 关大 Boss 胜利 | Boss 能力三选一 | 只在大 Boss 后给质变能力 |
