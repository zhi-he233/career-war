# 肉鸽地图规则

## 当前程序规则草案

| 项目 | 当前规则 | 说明 |
| --- | --- | --- |
| 可见未来层 | 当前关后预生成 10 层 | UI 只渲染视口附近节点 |
| 层节点数 | 以 2-3 个为主 | 起点和部分收束层为 1 个 |
| 4 节点层 | 暂不作为主要节奏 | 后续可少量加入 |
| 单节点连接数 | 最多 2 条 | 保持路线清楚 |
| 相邻层总连线 | 约 3-5 条 | 避免一团线 |
| Boss 节奏 | 作为远处目标出现 | 不严格每 3 关一个 |
| 路线目标 | 左 / 中 / 右尽量清楚 | 让玩家一眼看懂风险 |

## 推荐节奏

| 层类型 | 节点数量 | 推荐房间 | 说明 |
| --- | --- | --- | --- |
| 起点层 | 1 | normal | 初始当前节点 |
| 普通层 | 2 | normal / event / shop | 简单选择 |
| 分叉层 | 3 | normal / elite / event | 主要路线选择 |
| 收束层 | 1-2 | rest / reward / normal | 减少混乱 |
| 风险层 | 2-3 | elite / boss / normal | 给高收益路线 |
| Boss 层 | 1 | boss | 远处目标 |

## 地图路线建议表

| 阶段 | 推荐节点数 | 推荐房间组合 | 设计目的 | 备注 |
| --- | --- | --- | --- | --- |
| early | 2 | normal / event | 教学和轻量选择 | 不要惩罚太重 |
| early | 3 | normal / elite / shop | 第一次风险收益 | 精英奖励要明显 |
| mid | 2 | rest / reward | 缓冲和构筑整理 | 放在高压后 |
| mid | 3 | normal / elite / event | 形成路线差异 | 事件可带代价 |
| late | 1 | boss | 阶段目标 | Boss 主题要强 |
